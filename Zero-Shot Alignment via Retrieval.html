§1 The setup

Per-user fine-tuning doesn't scale.

Labelled per-user data, VRAM, training loops, and a shared base model that silently drifts with every new tenant. We want style alignment to be a retrieval problem, not a training problem.
A · per-user fine-tune doesn't scale
N users → N fine-tuned models
Cost scales with users. The base drifts.
B · retrieval-composed styles zero-shot
N users → one frozen base + shared style bank
One base. One bank. Cost constant in users.

§2 Retrieval + Knowledge / Style / Judge

Retrieve a style card, then run the KSJ loop.

Ollama for all three roles. Encode preference with all-MiniLM-L6-v2, FAISS IndexFlatIP picks top-K. Four action signals; MAX_REVISIONS = 2; always emit the best candidate seen. No LLM training — the modules are pre-authored style cards.
Style card · one file per style
{
  "id": "casual_friendly",
  "tags": ["casual", "warm"],
  "instruction": "conversational, contractions…",
  "examples": [ …2 prompts+answers… ]
}
Four action edges
  • 01accept → emit & return
  • 02revise_style → harder prompt, keep card
  • 03content_drift → re-emphasize draft facts
  • 04wrong_style → advance to top-K next

§3 Rationale

Each role is load-bearing.

Every design choice kills a specific failure mode we saw in the single-LLM baseline. Each is one env var or one file away from being swapped.
01 — DECOUPLING
Separate draft from restyle.
prevents →
Style LLM hallucinating facts to appear more stylistic when it has to draft and stylize in the same pass.
Knowledge & Style LLMs · judge/agents/knowledge.py · style.py
02 — CROSS-FAMILY JUDGE
Judge on a different model family.
prevents →
Self-preference bias: a judge from family F systematically over-scores F's own outputs.
Zheng et al. 2023 · env : JUDGE_MODEL=mistral:7b-instruct
03 — LOCAL COSINE
Content check outside the judge LLM.
prevents →
Hallucinations that slip past the judge's LLM call but that sentence embeddings catch.
CONTENT_PRESERVATION_MIN = 0.70 · sentence-transformers
04 — BOUNDED REVISION
Cap the loop at MAX_REVISIONS.
prevents →
Pathological prompts looping forever; unbounded inference cost under adversarial input.
MAX_REVISIONS = 2 · always-emit-best-seen
